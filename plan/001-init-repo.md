# Plan: Initialize Corrode Repository

## Context

Fresh repo with 1 commit on `master`. Submodules not initialized,
no `.gitmodules`, no `.gitignore`, no project `CLAUDE.md`.

## Steps

### 1. Add `.gitignore`

Embedded C++/Daisy project ignores:

- `build/` (compiled artifacts)
- `*.o`, `*.d`, `*.elf`, `*.bin`, `*.hex`, `*.map`
- `.vscode/` (user-specific)
- Submodule build artifacts

### 2. Add git submodules

```bash
git submodule add https://github.com/electro-smith/libDaisy.git \
  lib/libDaisy
git submodule add https://github.com/electro-smith/DaisySP.git \
  lib/DaisySP
```

Creates `.gitmodules` and populates `lib/`.

### 3. Create project `CLAUDE.md`

Full setup including:

- Project description (TB-303 synth, Daisy Patch Init, C++17)
- Build commands (`make`, `make program-dfu`, `make clean`,
  `make DEBUG=1`)
- Toolchain prerequisites (ARM GCC cross-compiler)
- Submodule build steps (libDaisy, DaisySP)
- Test commands (compile+run test files in `test/`)
- Project structure overview (src/, dsp/, lib/, test/, docs/)
- Key conventions (single-precision float DSP, no raw pointers,
  constexpr, block processing)

### 4. Commit and push

Single commit with all init changes (`.gitignore`, submodules,
`CLAUDE.md`).

## Files created/modified

- `.gitignore` (new)
- `.gitmodules` (auto-generated by `git submodule add`)
- `CLAUDE.md` (new)
- `lib/libDaisy/` (submodule)
- `lib/DaisySP/` (submodule)

## Verification

1. `git log --oneline` shows 2 commits on `master`
2. `git submodule status` shows both submodules initialized
3. `git remote -v` confirms origin tracking
4. `markdownlint-cli2 CLAUDE.md` passes
